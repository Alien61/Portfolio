Mise en place d'un VPN WireGuard

Principe d'un VPN

Un VPN (Virtual Private Network) crée un tunnel chiffré entre un client distant et un réseau interne. Tout le trafic passe par ce tunnel comme si le client était physiquement dans le réseau local.

Cas d'usage typique :

  • Un commercial en télétravail accède aux serveurs internes de l'entreprise
  • Un admin se connecte depuis chez lui pour maintenir les serveurs
  • Sécuriser un poste sur un Wi-Fi public (hôtel, aéroport)

Pourquoi WireGuard

Comparé à OpenVPN ou IPsec, WireGuard offre :

  • Un code très réduit (~4000 lignes vs ~600 000 pour OpenVPN) → moins de failles
  • Des performances supérieures (débit, latence)
  • Une configuration simple (quelques lignes)
  • Une cryptographie moderne (Curve25519, ChaCha20, Poly1305)
  • Intégré nativement au kernel Linux depuis la version 5.6

Architecture cible

┌──────────────┐        Internet       ┌──────────────┐
│  Client VPN  │ ◄────── chiffré ─────► │ Serveur VPN  │
│  (laptop)    │      port UDP 51820    │  (Debian)    │
│ 10.8.0.2/24  │                        │ 10.8.0.1/24  │
└──────────────┘                        └──────┬───────┘
                                               │
                                               ▼
                                     ┌─────────────────┐
                                     │ Réseau interne  │
                                     │  10.0.0.0/24    │
                                     └─────────────────┘

Plan d'adressage du tunnel :

  • Réseau VPN : 10.8.0.0/24
  • Serveur : 10.8.0.1
  • Premier client : 10.8.0.2

Installation sur le serveur Debian

sudo apt update
sudo apt install wireguard wireguard-tools -y

# Vérifier l'installation
wg --version

Génération des clés côté serveur

cd /etc/wireguard
umask 077

# Clé privée du serveur
wg genkey | tee server_private.key | wg pubkey > server_public.key

# Afficher les deux
cat server_private.key
cat server_public.key
🔐

umask 077 garantit que les fichiers créés ne soient lisibles que par root. La clé privée ne doit JAMAIS être partagée.

Fichier de configuration wg0.conf

sudo nano /etc/wireguard/wg0.conf
[Interface]
# Adresse du serveur dans le tunnel
Address = 10.8.0.1/24
ListenPort = 51820
PrivateKey = <contenu de server_private.key>

# NAT vers le LAN + activation du forwarding
PostUp = iptables -A FORWARD -i %i -j ACCEPT
PostUp = iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
PostDown = iptables -D FORWARD -i %i -j ACCEPT
PostDown = iptables -t nat -D POSTROUTING -o eth0 -j MASQUERADE

# Premier peer (client) — à compléter après génération des clés client
[Peer]
PublicKey = <clé publique du client>
AllowedIPs = 10.8.0.2/32
💡

Remplacer eth0 par le nom réel de l'interface réseau du serveur (vérifier avec ip a).

Activer le routage IP sur le serveur

Pour que le serveur fasse passer le trafic du VPN vers le LAN :

sudo nano /etc/sysctl.conf

Décommenter ou ajouter :

net.ipv4.ip_forward=1

Appliquer :

sudo sysctl -p

Ouvrir le port UDP 51820 sur le firewall

sudo ufw allow 51820/udp
sudo ufw allow ssh   # ne pas se bloquer soi-même !
sudo ufw enable
sudo ufw status

Démarrer le serveur WireGuard

# Lancer le tunnel
sudo wg-quick up wg0

# Activer au démarrage
sudo systemctl enable wg-quick@wg0

# Vérifier le statut
sudo wg show

Configuration du client (Windows / Linux)

Installer le client officiel WireGuard : wireguard.com/install

Générer les clés du client

Sur Linux :

wg genkey | tee client_private.key | wg pubkey > client_public.key

Sur Windows, le client graphique génère automatiquement les clés à la création du tunnel.

Fichier client.conf

[Interface]
PrivateKey = <clé privée du client>
Address = 10.8.0.2/24
DNS = 10.0.0.10

[Peer]
PublicKey = <clé publique du serveur>
Endpoint = <IP publique du serveur>:51820
AllowedIPs = 10.0.0.0/24, 10.8.0.0/24
PersistentKeepalive = 25

Côté serveur — ajouter la clé publique du client

Dans /etc/wireguard/wg0.conf, compléter la section [Peer] :

[Peer]
PublicKey = <clé publique du client>
AllowedIPs = 10.8.0.2/32

Recharger la configuration :

sudo wg-quick down wg0
sudo wg-quick up wg0

Test & vérification

Sur le client, activer le tunnel (bouton Activate dans WireGuard GUI).

Vérifier la connexion :

# Pinger le serveur VPN
ping 10.8.0.1

# Pinger une machine du LAN de l'entreprise
ping 10.0.0.10

# Vérifier l'IP publique vue depuis Internet
curl ifconfig.me

Côté serveur :

sudo wg show

La sortie doit montrer un latest handshake récent et des transferts de données.

✅

Le client accède désormais aux ressources internes 10.0.0.0/24 comme s'il était physiquement dans les locaux, via un tunnel chiffré. Cette procédure répond directement au cas d'usage BTS : "La PME envisage d'utiliser un VPN pour ses télétravailleurs."
