Objectif du projet

Déployer une infrastructure réseau d'entreprise complète sur Windows Server. Cette infrastructure doit permettre :

  • L'authentification centralisée des utilisateurs via Active Directory
  • L'organisation logique de la société par Unités d'Organisation (OU)
  • La distribution automatique des adresses IP via DHCP
  • L'application de stratégies de groupe (GPO) de sécurité
  • Le chiffrement des communications par certificats

Paramétrage IP du serveur

Un contrôleur de domaine doit obligatoirement avoir une IP statique. Si l'IP change, les clients ne trouveront plus le DC.

Configuration statique

  • Panneau de configuration → Centre Réseau et partage
  • Clic droit sur la carte réseau → Propriétés
  • Sélectionner IPv4 → Propriétés
Adresse IP       : 10.0.0.10
Masque           : 255.255.255.0
Passerelle       : 10.0.0.254
DNS primaire     : 127.0.0.1   (le serveur est son propre DNS)
💡

Renommer le serveur avant la promotion en DC (ex : SRV-AD01) car changer le nom après implique de re-promouvoir le serveur.

Installation AD DS & création du domaine

Étape 1 — Ajouter le rôle

  • Ouvrir Gestionnaire de serveur
  • Gérer → Ajouter des rôles et fonctionnalités
  • Cocher Services AD DS + Serveur DNS
  • Valider et attendre la fin de l'installation

Étape 2 — Promotion en contrôleur de domaine

  • Cliquer sur le drapeau ⚑ en haut à droite → Promouvoir ce serveur en contrôleur de domaine
  • Choisir Ajouter une nouvelle forêt
  • Nom de domaine racine : alienis.lan
  • Niveau fonctionnel : Windows Server 2016 ou plus
  • Mot de passe DSRM : à conserver précieusement
  • Laisser par défaut le reste → Installer
⚠️

Le serveur va redémarrer automatiquement après la promotion. La connexion se fait désormais en ALIENIS\Administrateur.

Création de l'OU Société

Les Unités d'Organisation (OU) structurent l'annuaire et permettent d'appliquer des GPO de façon ciblée.

  • Ouvrir Utilisateurs et ordinateurs Active Directory
  • Clic droit sur le domaine alienis.lan → Nouveau → Unité d'organisation
  • Nom : SOCIÉTÉ
  • Cocher Protéger de la suppression accidentelle

Sous-OU à créer dans SOCIÉTÉ

  • Utilisateurs — comptes des employés
  • Ordinateurs — postes de travail du domaine
  • Groupes — groupes de sécurité
  • Services — comptes de service

Ajout des postes clients au domaine

Sur le poste client (Windows 10/11 Pro) :

Étape 1 — Configurer l'IP du client

Étape 3 — Rejoindre le domaine

  • Paramètres → Système → À propos
  • Renommer ce PC (avancé) → onglet Nom de l'ordinateur → Modifier

  • Authentification avec un compte Administrateur du domaine
  • Redémarrer le PC

Configuration du DHCP & bail

Étape 1 — Ajouter le rôle DHCP

  • Gestionnaire de serveur → Ajouter des rôles → Serveur DHCP
  • Après installation, cliquer sur le drapeau → Terminer la configuration DHCP
  • Autoriser avec le compte Administrateur du domaine

Étape 2 — Créer une étendue IPv4

GPO Autorité de certification

Cette GPO permet de déployer un certificat d'autorité racine sur tous les postes du domaine, indispensable pour les communications chiffrées (SMB signing, HTTPS interne).

Étape 1 — Installer AD CS

  • Ajouter le rôle Services de certificats Active Directory
  • Configurer l'AC comme AC racine d'entreprise
  • Validité : 10 ans

Étape 2 — Créer la GPO

  • Ouvrir Gestion des stratégies de groupe
  • Créer un dossier SOCIÉTÉ → clic droit → Créer un objet GPO
  • Nommer : GPO-Certificats-AC
  • Modifier → Configuration ordinateur → Stratégies → Paramètres Windows → Paramètres de sécurité → Stratégies de clé publique
  • Importer le certificat de l'AC racine

Étape 3 — Lier la GPO

gpupdate /force
certutil -pulse

GPO — Désactiver le gestionnaire des tâches

Empêcher les utilisateurs standards de fermer des processus critiques du poste.

Création de la GPO

  • Dans Gestion des stratégies de groupe
  • Clic droit sur l'OU SOCIÉTÉ > Utilisateurs → Créer un objet GPO dans ce domaine, et le lier ici
  • Nommer : GPO-Restriction-TaskMgr

Paramétrage

Modifier → Configuration utilisateur → Modèles d'administration → Système → Options Ctrl+Alt+Suppr

  • Activer : Supprimer le Gestionnaire des tâches
gpupdate /force
⚠️

Bien cibler l'OU des utilisateurs et non celle des administrateurs, sinon les admins du domaine seront également bloqués.

Vérifications finales

Tests à effectuer

  • ✅ Un client peut s'authentifier avec un compte AD
  • ✅ Le client reçoit bien une IP du DHCP (vérifier dans les baux côté serveur)
  • ✅ Le client résout alienis.lan (nslookup alienis.lan)
  • ✅ Le certificat d'AC est présent dans le magasin du client
  • ✅ Le Gestionnaire des tâches est bien bloqué pour les users standards
  • ✅ Les admins ont toujours accès au Gestionnaire

Commandes utiles pour l'audit

Get-ADUser -Filter * | Select Name, SamAccountName
Get-ADComputer -Filter * | Select Name, OperatingSystem
Get-DhcpServerv4Lease -ScopeId 10.0.0.0
Get-GPO -All | Select DisplayName, GpoStatus
