Création d'un Active Directory + Client

Objectif & principe

L'Active Directory (AD) est le service d'annuaire Microsoft qui gère l'authentification et l'identification des utilisateurs et des ressources dans un réseau d'entreprise.

Il centralise la gestion :

  • Des comptes utilisateurs (login unique pour toute l'entreprise)
  • Des groupes et des permissions
  • Des ressources partagées : dossiers, imprimantes, applications
  • Des politiques de sécurité via GPO

Prérequis

  • Une VM Windows Server 2022 (ou 2019) installée
  • Un compte Administrateur local
  • Connexion réseau fonctionnelle
  • Une VM Windows 10/11 Pro pour le client (Home ne peut pas rejoindre un domaine)

IP statique du serveur

Un contrôleur de domaine doit avoir une IP fixe. Si elle change, les clients ne trouveront plus le DC.

  • Paramètres → Réseau et Internet → Paramètres réseau avancés
  • Clic droit sur la carte → Propriétés → IPv4 → Propriétés
Adresse IP    : 10.0.0.10
Masque        : 255.255.255.0
Passerelle    : 10.0.0.254
DNS primaire  : 127.0.0.1

Renommer le serveur (avant promotion)

⚠️

Toujours renommer avant la promotion en DC : changer le nom après implique une reconfiguration complète.

Rename-Computer -NewName "SRV-AD01" -Restart

Installation du rôle AD DS

  • Ouvrir le Gestionnaire de serveur
  • Menu Gérer → Ajouter des rôles et fonctionnalités
  • Type d'installation : Installation basée sur un rôle
  • Sélectionner le serveur local
  • Cocher Services AD DS (accepter les fonctionnalités associées)
  • Cocher Serveur DNS si pas déjà présent
  • Valider l'installation et attendre la fin

Promotion en contrôleur de domaine

Après l'installation du rôle, une notification apparaît (drapeau ⚑) :

  • Cliquer sur le drapeau → Promouvoir ce serveur en contrôleur de domaine
  • Sélectionner Ajouter une nouvelle forêt
  • Nom du domaine racine : alienis.lan
  • Niveau fonctionnel : Windows Server 2016 ou plus récent
  • Saisir un mot de passe DSRM (mode restauration)
  • Laisser le reste par défaut
  • Cliquer sur Installer → le serveur redémarre

Vérification côté serveur

Après redémarrage, se connecter avec ALIENIS\Administrateur.

# Vérifier le rôle
Get-ADDomain

# Voir le nom du contrôleur
Get-ADDomainController

# Tester le DNS
nslookup alienis.lan

Les outils d'administration sont disponibles dans Outils d'administration Windows :

  • Utilisateurs et ordinateurs Active Directory
  • Domaines et approbations Active Directory
  • Sites et services Active Directory
  • Gestion des stratégies de groupe

Configuration IP du poste client

Sur la VM Windows 10/11 Pro, configurer une IP statique :

Adresse IP    : 10.0.0.50
Masque        : 255.255.255.0
Passerelle    : 10.0.0.254
DNS primaire  : 10.0.0.10     ← IP du serveur AD
DNS secondaire: (vide ou DNS public)
⚠️

Le DNS primaire du client doit absolument être l'IP du contrôleur de domaine. Sans ça, la jointure au domaine échoue.

Tester la résolution du domaine :

ping alienis.lan
nslookup alienis.lan

Jointure du client au domaine

  • Paramètres → Système → À propos
  • Section Paramètres associés → Renommer ce PC (avancé)
  • Onglet Nom de l'ordinateur → bouton Modifier
  • Cocher Domaine → saisir alienis.lan
  • S'authentifier avec un compte Administrateur du domaine
  • Message de bienvenue → OK → redémarrer le PC

Au redémarrage, se connecter avec un compte du domaine :

Utilisateur : ALIENIS\Administrateur   (ou j.dupont@alienis.lan)
Mot de passe: ****
✅

Le PC apparaît désormais dans Utilisateurs et ordinateurs AD → Computers côté serveur. Les comptes AD peuvent s'y connecter.
