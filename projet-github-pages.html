Projet 01 — Web

Mise en ligne via GitHub Pages

Héberger gratuitement un site web statique sur GitHub Pages, avec déploiement automatique à chaque git push.

EnvironnementGitHub · Git CLI
Durée~30 minutes
CoûtGratuit
NiveauDébutant
Sommaire

Objectif

Publier un site web statique (HTML/CSS/JS) en ligne, accessible à l'URL https://<username>.github.io/<repo>, sans payer d'hébergement et avec déploiement automatique à chaque modification.

Prérequis

  • Un compte GitHub (gratuit)
  • Git installé localement → git --version
  • Un terminal (PowerShell / Bash / Zsh)
  • Les fichiers du site (index.html, style.css, etc.)

Création du repository GitHub

  • Se connecter sur github.com
  • Cliquer sur New (bouton vert en haut à droite)
  • Nom du repo : portfolio (ou tout autre nom)
  • Visibilité : Public (obligatoire pour GitHub Pages gratuit)
  • Ne pas initialiser avec README si le projet local existe déjà
  • Cliquer sur Create repository

Configuration Git locale

Si Git n'a jamais été configuré :

git config --global user.name "VotreNom"
git config --global user.email "vous@exemple.com"

Se placer dans le dossier du site et initialiser :

cd /chemin/vers/le/site
git init
git branch -M main
git remote add origin https://github.com/VotreNom/portfolio.git

Premier push

git add .
git commit -m "Initial commit — portfolio"
git push -u origin main
🔐

Depuis 2021, GitHub n'accepte plus le mot de passe pour Git. Il faut utiliser un Personal Access Token (PAT) → github.com/settings/tokens → scope repo.

Activation de GitHub Pages

  • Sur le repository GitHub → onglet Settings
  • Menu latéral → Pages
  • Source : Deploy from a branch
  • Branch : main / dossier / (root)
  • Cliquer sur Save

Après 1 à 2 minutes, le site est accessible à :

https://VotreNom.github.io/portfolio/

Domaine personnalisé (optionnel)

Pour utiliser www.mondomaine.com à la place de l'URL GitHub :

Côté GitHub

  • Settings → Pages → Custom domain → saisir www.mondomaine.com
  • Attendre la validation DNS (quelques minutes à 24h)
  • Cocher Enforce HTTPS

Workflow quotidien

Pour mettre à jour le site après une modification :

# Voir les fichiers modifiés
git status

# Stager les modifs
git add .

# Commit avec message descriptif
git commit -m "Update : nouveau projet VPN"

# Envoyer sur GitHub
git push

Dès que le push est effectué, GitHub Actions déploie automatiquement la nouvelle version. Le site est mis à jour en 30 secondes à 2 minutes.

✅

Le site est désormais en production. Chaque push déclenche un nouveau déploiement visible dans l'onglet Actions du repo.
