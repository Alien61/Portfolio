Présentation du projet

AutoPage est un système d'automatisation qui génère et publie automatiquement des pages HTML sur un portfolio GitHub Pages à partir de contenu rédigé dans Notion. Dès qu'une nouvelle entrée est créée dans une base de données Notion, le workflow s'enclenche sans aucune intervention manuelle.

Prérequis

  • Docker Desktop installé sur Windows
  • Un compte GitHub avec un repository Portfolio
  • Un compte Notion avec un workspace
  • Un navigateur web (Edge ou Firefox)
  • PowerShell disponible

Configuration Docker Compose

Créer un dossier de travail, par exemple :

C:\Users\alien\Desktop\informatique\Portfolio\projet automatisation

Créer un fichier docker-compose.yml :

services:
  n8n:
    image: docker.n8n.io/n8nio/n8n:latest
    container_name: n8n
    user: root
    restart: unless-stopped
    ports:
      - "5678:5678"
    environment:
      - GENERIC_TIMEZONE=Europe/Paris
      - TZ=Europe/Paris
      - N8N_EDITOR_BASE_URL=http://localhost:5678
      - WEBHOOK_URL=http://localhost:5678
      - N8N_PROTOCOL=http
      - N8N_HOST=localhost
      - N8N_PORT=5678
      - GIT_USER_EMAIL=votre-email@example.com
      - GIT_USER_NAME=VotreNomGitHub
      - N8N_USER_FOLDER=/home/node
    volumes:
      - n8n_data:/home/node
      - portfolio_repo:/home/node/portfolio
      - n8n_ssh:/root/.ssh
    depends_on:
      - ollama
      - ssh
    networks:
      - ai_network

  ollama:
    image: ollama/ollama:latest
    container_name: ollama
    restart: unless-stopped
    ports:
      - "11434:11434"
    volumes:
      - ollama_data:/root/.ollama
    networks:
      - ai_network

  ssh:
    image: lscr.io/linuxserver/openssh-server:latest
    container_name: ssh_server
    restart: unless-stopped
    ports:
      - "2222:2222"
    environment:
      - PUID=0
      - PGID=0
      - USER_NAME=n8nuser
      - USER_PASSWORD=root123
      - PASSWORD_ACCESS=true
    volumes:
      - portfolio_repo:/home/node/portfolio
      - ssh_server_keys:/config/ssh_host_keys
      - ssh_authorized:/root/.ssh
    networks:
      - ai_network

volumes:
  n8n_data:
  ollama_data:
  portfolio_repo:
  n8n_ssh:
  ssh_server_keys:
  ssh_authorized:

networks:
  ai_network:
    driver: bridge
💡

Remplacer votre-email@example.com et VotreNomGitHub par les vraies informations GitHub. Les volumes n8n_ssh, ssh_server_keys et ssh_authorized persistent les clés SSH entre redémarrages.

Lancement des containers

cd "C:\Users\alien\Desktop\informatique\Portfolio\projet automatisation"
docker compose up -d
docker ps

Le résultat doit montrer trois containers avec le statut Up : n8n, ollama, ssh_server.

Installation du modèle Ollama

docker exec -it ollama ollama pull llama3
⏱️

Le modèle pèse environ 4,7 Go. La génération de pages tourne sur CPU et prend entre 3 et 10 minutes par page.

Installer Git et cloner le repo

docker exec -it ssh_server sh
apk add --no-cache git

git config --global user.email "votre-email@example.com"
git config --global user.name "VotreNomGitHub"

cd /home/node/portfolio
git clone https://github.com/VotreNomGitHub/Portfolio.git .
git config --global --add safe.directory /home/node/portfolio
exit
⚠️

Git doit être réinstallé à chaque recréation du container (Alpine Linux ne l'inclut pas). Les volumes persistent les clés mais pas les paquets installés.

Configuration SSH par clé publique

Générer les clés dans n8n

docker exec -it n8n sh
ssh-keygen -t rsa -b 4096 -f /root/.ssh/n8n_key -N ""
cat /root/.ssh/n8n_key.pub
exit

Déposer la clé dans ssh_server

docker exec -it ssh_server sh
mkdir -p /root/.ssh
echo "COLLER_LA_CLE_PUBLIQUE_ICI" > /root/.ssh/authorized_keys
chmod 700 /root/.ssh
chmod 600 /root/.ssh/authorized_keys
exit

Récupérer l'IP du container ssh_server

docker inspect ssh_server | grep IPAddress

Création du token GitHub

  • Aller sur https://github.com/settings/tokens
  • Generate new token (classic)
  • Nom : n8n-portfolio
  • Expiration : No expiration
  • Cocher uniquement : repo
  • Copier le token (affiché une seule fois !)
🔐

Ne jamais partager ce token publiquement ni le committer dans le repository. Le stocker dans un endroit sûr.

Workflow n8n — 10 nodes

Ouvrir n8n à http://localhost:5678. Créer un compte, puis un nouveau workflow.

Node 1 — Notion Trigger

EventPage added to database
Poll TimeEvery Minute (test) / Every 15 Minutes (prod)
Retry On FailActivé

Node 2 — Notion Get Page

Resource : Page — Operation : Get a database page — Page ID : {{ $json.id }} (expression).

Node 3 — Code (JavaScript)

const page = $input.first().json;
const titre = page.property_titre || '';
const description = page.property_description || '';
const contenu = page.property_contenu || '';
const slug = page.property_slug || titre.toLowerCase()
  .replace(/\s+/g, '-').replace(/[^a-z0-9-]/g, '');
const tags = Array.isArray(page.property_tags)
  ? page.property_tags.join(', ') : '';

const htmlHeader = `<!DOCTYPE html>...nav...`;
const htmlFooter = `...footer</html>`;

const tagsHTML = tags.split(', ')
  .map(t => `<span class="skill-tag">${t.trim()}</span>`).join('\n');

const promptCarte = `Tu es un générateur HTML...
  TITRE: ${titre} | SLUG: ${slug} | TAGS: ${tags}`;

const promptPage = `Tu es un générateur HTML.
  Retourne UNIQUEMENT le code entre <section> et </section>.
  RÈGLES: chaque partie = <div class="card">, listes = <ul><li>,
  commandes = <pre><code>, TITRE: ${titre} | CONTENU: ${contenu}`;

return [{ json: {
  titre, description, contenu, slug, tags, htmlHeader, htmlFooter, tagsHTML,
  ollamaBodyCarte: JSON.stringify({ model: 'llama3', stream: false, prompt: promptCarte }),
  ollamaBodyPage: JSON.stringify({ model: 'llama3', stream: false, prompt: promptPage })
}}];

Nodes 4 & 5 — HTTP Request vers Ollama (parallèle)

MethodPOST
URLhttp://ollama:11434/api/generate
Body JSON (Node 4){{ $json.ollamaBodyCarte }}
Body JSON (Node 5){{ $json.ollamaBodyPage }}
Timeout600 000 ms (10 min)

Node 6 — Merge

Mode : Combine — Combine by : Position. Fusionne les deux réponses Ollama.

Node 7 — Code2 (assemblage)

const data = $('Code').first().json;
const rawCarte = $('HTTP Request').first().json.response || '';
const rawPage = $('HTTP Request2').first().json.response || '';

let section = rawPage.replace(/```html/gi,'').replace(/```/g,'').trim();
const start = section.indexOf('<section>');
const end = section.lastIndexOf('</section>');
if (start !== -1 && end !== -1) {
  section = section.substring(start, end + 10);
}

const fullPage = data.htmlHeader + '\n' + section + '\n' + data.htmlFooter;
const htmlCarte = rawCarte.replace(/```html/gi,'').replace(/```/g,'').trim();
const slug = data.slug;

// Encoder en base64 pour éviter les problèmes de caractères spéciaux
const fullPageB64 = Buffer.from(fullPage,'utf8').toString('base64').replace(/\n/g,'');
const htmlCarteEscaped = htmlCarte.replace(/\\/g,'\\\\').replace(/'/g,"'\\\\'");

const cmdEcrirePage = `echo '${fullPageB64}' | base64 -d > /home/node/portfolio/${slug}.html`;
const cmdInsertCarte = `printf '%s' '${htmlCarteEscaped}' > /tmp/carte.html
  && sed -i '/<!-- INSERTION AUTOMATIQUE ICI -->/ r /tmp/carte.html' /home/node/portfolio/projets.html`;

return [{ json: { htmlCarte, htmlPage: fullPage, slug, cmdEcrirePage, cmdInsertCarte } }];

Nodes 8, 9, 10 — SSH

Credentials SSH : Host = IP ssh_server, Port = 2222, User = n8nuser, Auth = Private Key (contenu de n8n_key).

  • Node 8 — Command : {{ $('Code2').first().json.cmdEcrirePage }}
  • Node 9 — Command : {{ $('Code2').first().json.cmdInsertCarte }}
  • Node 10 — Git commit & push :
cd /home/node/portfolio \
  && git config --global user.email "votre-email@example.com" \
  && git config --global user.name "VotreNomGitHub" \
  && git config --global --add safe.directory /home/node/portfolio \
  && git add . \
  && git commit -m "New project: {{ $('Code2').first().json.slug }}" \
  && git pull --rebase https://VotreNomGitHub:TOKEN@github.com/VotreNomGitHub/Portfolio.git main \
  && git push https://VotreNomGitHub:TOKEN@github.com/VotreNomGitHub/Portfolio.git

Test & activation en production

  1. Créer une nouvelle entrée dans la base Notion (titre, description, slug, contenu)
  2. Dans n8n, cliquer sur Test workflow
  3. Attendre 10 à 15 minutes (CPU)
  4. Vérifier sur GitHub : fichier ${slug}.html créé
  5. Vérifier projets.html : nouvelle carte insérée
  6. Attendre 1 à 2 minutes pour GitHub Pages
  7. Consulter https://VotreNomGitHub.github.io/Portfolio

Activation en production

  • Cliquer sur le bouton Inactive en haut à droite du workflow → Active
  • Passer le Poll Time du Notion Trigger à Every 15 Minutes
🎉

Le système tourne désormais en autonomie. Chaque nouvelle page Notion déclenche automatiquement la génération HTML, la mise à jour du portfolio et la publication sur GitHub Pages.
