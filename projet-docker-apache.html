Projet 03 — DevOps

Docker Apache2 + docker-compose avec SSH GitHub

Conteneurisation d'Apache2 avec docker-compose, connexion SSH via GitHub pour héberger un site web, avec tout le workflow Git.

StackDocker · Apache · SSH
Durée~1h30
Port8080 → 80
NiveauIntermédiaire

Objectif

Conteneuriser un serveur web Apache2 avec docker-compose, et synchroniser le contenu avec un repository GitHub via SSH. Le déploiement devient reproductible et portable sur n'importe quelle machine avec Docker.

Prérequis

  • Docker Desktop ou Docker Engine installé
  • docker --version et docker compose version fonctionnels
  • Un compte GitHub avec un repository contenant le site
  • PowerShell ou Bash

Fichier docker-compose.yml

Créer un dossier de travail et le fichier docker-compose.yml :

services:
  apache:
    image: httpd:2.4
    container_name: apache-web
    restart: unless-stopped
    ports:
      - "8080:80"
    volumes:
      - site_data:/usr/local/apache2/htdocs
    networks:
      - web_network

  ssh:
    image: lscr.io/linuxserver/openssh-server:latest
    container_name: ssh-web
    restart: unless-stopped
    ports:
      - "2222:2222"
    environment:
      - PUID=0
      - PGID=0
      - USER_NAME=deploy
      - PASSWORD_ACCESS=true
      - USER_PASSWORD=change-me
    volumes:
      - site_data:/usr/local/apache2/htdocs
      - ssh_keys:/root/.ssh
    networks:
      - web_network

volumes:
  site_data:
  ssh_keys:

networks:
  web_network:
    driver: bridge
💡

Le volume site_data est partagé entre les deux conteneurs : Apache le sert en lecture, SSH y écrit les mises à jour venant de GitHub.

Lancement des services

# Lancer les deux conteneurs en arrière-plan
docker compose up -d

# Vérifier l'état
docker ps

# Voir les logs
docker logs apache-web
docker logs ssh-web

Génération des clés SSH pour GitHub

Entrer dans le conteneur SSH :

docker exec -it ssh-web sh
apk add --no-cache git openssh-keygen
ssh-keygen -t ed25519 -f /root/.ssh/github_key -N ""
cat /root/.ssh/github_key.pub

Copier la clé publique et l'ajouter sur GitHub :

  • GitHub → Settings → SSH and GPG keys
  • New SSH key → coller la clé publique

Configurer SSH pour utiliser cette clé :

cat > /root/.ssh/config <<EOF
Host github.com
    HostName github.com
    User git
    IdentityFile /root/.ssh/github_key
EOF
chmod 600 /root/.ssh/config /root/.ssh/github_key

# Tester la connexion
ssh -T git@github.com

Clone du repository GitHub

cd /usr/local/apache2/htdocs
rm -rf ./*   # vider le contenu par défaut
git clone git@github.com:VotreNom/portfolio.git .
git config --global --add safe.directory /usr/local/apache2/htdocs

Workflow Git complet

Configuration initiale (une fois)

git config --global user.email "vous@exemple.com"
git config --global user.name "VotreNom"

Mise à jour du site depuis GitHub

cd /usr/local/apache2/htdocs
git pull --rebase origin main

Modification directe dans le conteneur

cd /usr/local/apache2/htdocs
# ... modifier les fichiers ...
git status
git add .
git commit -m "Update: contenu mis à jour"
git pull --rebase origin main   # toujours avant le push
git push origin main
⚠️

Toujours faire git pull --rebase avant git push pour éviter les rejets si le remote a des commits plus récents.

Gestion des conflits

# En cas de conflit lors du pull
git status                    # identifier les fichiers en conflit
# ... résoudre manuellement les conflits dans les fichiers ...
git add <fichier-résolu>
git rebase --continue

# Ou annuler le rebase
git rebase --abort

Accès au site

Le site est accessible à :

http://localhost:8080

Depuis une autre machine du réseau :

http://<IP-de-la-machine-hôte>:8080

Arrêt & redémarrage

# Arrêter sans perdre les données
docker compose stop

# Redémarrer
docker compose start

# Tout supprimer (conteneurs + volumes)
docker compose down -v
✅

Les modifications dans le repo GitHub peuvent être déployées instantanément avec un simple git pull dans le conteneur — un vrai workflow CI/CD minimal.
