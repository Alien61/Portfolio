Mise en place du DHCP IPv4

Principe du DHCP

Le DHCP (Dynamic Host Configuration Protocol) attribue automatiquement les paramètres réseau aux machines clientes :

  • Adresse IP
  • Masque de sous-réseau
  • Passerelle par défaut
  • Serveurs DNS
  • Nom de domaine

Sans DHCP, un administrateur devrait configurer chaque poste manuellement — impossible sur un réseau de plusieurs dizaines de machines.

Fonctionnement — les 4 étapes DORA

  1. DISCOVER — le client cherche un serveur DHCP (broadcast)
  2. OFFER — le serveur propose une adresse
  3. REQUEST — le client accepte l'offre
  4. ACK — le serveur confirme et crée le bail

Prérequis

  • Windows Server 2022 (ou 2019) installé
  • IP statique configurée sur le serveur
  • Réseau isolé (pas de box qui ferait déjà du DHCP) — sinon conflit
  • Idéalement dans un environnement AD déjà opérationnel
⚠️

Deux serveurs DHCP sur le même réseau physique = chaos garanti. Désactiver impérativement le DHCP de la box si on travaille sur le réseau domestique.

IP statique du serveur

Un serveur DHCP doit avoir une IP fixe.

  • Paramètres → Réseau et Internet → Paramètres réseau avancés
  • (ou l'ancien Centre Réseau et partage selon la version)
  • Propriétés de la carte → IPv4 → Propriétés
Adresse IP    : 10.0.0.10
Masque        : 255.255.255.0
Passerelle    : 10.0.0.254
DNS primaire  : 127.0.0.1

Installation du rôle DHCP

  • Ouvrir le Gestionnaire de serveur
  • Gérer → Ajouter des rôles et fonctionnalités
  • Type : Installation basée sur un rôle
  • Sélectionner le serveur local
  • Cocher Serveur DHCP (accepter les fonctionnalités associées)
  • Valider et installer

Autorisation dans Active Directory

Dans un environnement AD, un serveur DHCP doit être autorisé pour pouvoir répondre aux requêtes (mesure de sécurité contre les DHCP pirates).

  • Après l'installation, cliquer sur le drapeau ⚑ en haut du Gestionnaire
  • Cliquer sur Terminer la configuration DHCP (ou Configurer DHCP selon la version)
  • S'authentifier avec un compte Administrateur du domaine
  • Créer les groupes de sécurité DHCP (administrateurs, utilisateurs)

Création d'une étendue IPv4

  • Ouvrir la console DHCP depuis les outils d'administration
  • Déplier le serveur → clic droit sur IPv4 → Nouvelle étendue

Paramétrage de l'étendue

ParamètreValeur recommandée
Nom de l'étendueLAN-SOCIÉTÉ
DescriptionPostes clients du LAN
Adresse IP de début10.0.0.100
Adresse IP de fin10.0.0.200
Masque255.255.255.0 (/24)
Exclusions10.0.0.100 → 10.0.0.110 (réservées)
Durée du bail8 jours
💡

La plage d'exclusion empêche le DHCP de distribuer des IP qu'on veut garder pour les serveurs, imprimantes réseau, etc.

Options DHCP (passerelle, DNS)

Les options indiquent au client quelle passerelle et quel DNS utiliser.

CodeNomValeur
003Routeur (passerelle)10.0.0.254
006Serveurs DNS10.0.0.10
015Nom de domaine DNSalienis.lan
044Serveurs WINS (optionnel)—

Pour les activer :

  • Dans la console DHCP → étendue → Options d'étendue
  • Clic droit → Configurer les options
  • Cocher et renseigner les options 003, 006, 015

Activer l'étendue (si ce n'est pas déjà fait) :

  • Clic droit sur l'étendue → Activer

Réservation d'adresse par MAC

Pour qu'un poste spécifique reçoive toujours la même IP (imprimante, serveur applicatif…) :

Étape 1 — Récupérer l'adresse MAC du client

ipconfig /all

Chercher la ligne Adresse physique (format AA-BB-CC-DD-EE-FF).

Étape 2 — Créer la réservation

  • Dans la console DHCP → étendue → Réservations
  • Clic droit → Nouvelle réservation
  • Saisir :
    • Nom : PC-COMPTA-01
    • Adresse IP : 10.0.0.51
    • Adresse MAC : AA-BB-CC-DD-EE-FF (ou sans tirets)
    • Type : DHCP uniquement
💡

Autre méthode rapide : dans Pool d'adresses, clic droit sur un bail existant → Ajouter à la réservation.

Configuration du poste client

Passer le client en mode automatique :

  • Propriétés de la carte réseau → IPv4 → Propriétés
  • Cocher Obtenir une adresse IP automatiquement
  • Cocher Obtenir les adresses des serveurs DNS automatiquement

Forcer une demande de bail :

ipconfig /release
ipconfig /renew
ipconfig /all

Vérifier dans la sortie :

  • DHCP activé : Oui
  • Serveur DHCP : 10.0.0.10
  • Bail obtenu / Bail expirant : dates cohérentes

Vérification des baux côté serveur

  • Console DHCP → étendue → Baux d'adresses
  • Les clients qui ont obtenu une IP apparaissent avec nom, IP, MAC, expiration

PowerShell (équivalent rapide)

# Lister les baux
Get-DhcpServerv4Lease -ScopeId 10.0.0.0

# Lister les réservations
Get-DhcpServerv4Reservation -ScopeId 10.0.0.0

# Statistiques de l'étendue
Get-DhcpServerv4ScopeStatistics -ScopeId 10.0.0.0
✅

Le serveur DHCP est désormais opérationnel. Chaque poste connecté au réseau reçoit automatiquement une IP dans la plage définie, avec passerelle et DNS corrects.
